SLIDE 6: TOKENIZATION PIPELINE - 17 STAGES
===========================================
Title: "From Clinical Events to Model-Ready Tokens"

Visual: Horizontal pipeline with key metrics

INPUT: 14,283 MEDS events → OUTPUT: 17,429 tokens (72 unique token IDs)

Key Stages (grouped by function):

📊 FILTERING (Stages 1, 8, 15)
• Remove rare clinical codes (appearing <5 times)
• 892 codes → 245 codes retained

🔄 PREPROCESSING (Stages 2-5, 9-13)
• Transform ICD codes to hierarchical format (J96.00 → 3-6//J96, SFX//J96.00)
• Process diagnoses, procedures, admissions, demographics
• Extract patient static data (gender, birth year)

📈 QUANTIZATION (Stages 9-11)
• Convert numeric lab values to categorical bins
• (Not applicable to sample: no lab data)

⏰ TEMPORAL MODELING (Stages 16-17)
• Inject time interval tokens (=2d, =6h, =1mt between events)
• Calculate temporal statistics for prediction
• 11,652 events → 17,429 tokens (added 5,777 interval markers)

🗂️ VOCABULARY BUILDING (Stages 6-7)
• Count code frequencies across train/test
• Assign unique integer IDs to each code
• Final vocabulary: 72 tokens

Result: Patient timelines converted to integer sequences ready for transformer model

SLIDE 7: TECHNICAL CHALLENGES & SOLUTIONS
==========================================
Title: "Adaptive Problem-Solving: Dataset-Specific Modifications"

Present as problem → solution pairs with icons:

⚠️ CHALLENGE 1: Configuration Mismatch
Problem: MEDS extraction expected 11 tables, sample had only 5
Solution: Created simplified event_configs-sample.yaml with available tables only
Impact: Successful MEDS extraction without unnecessary dependencies

⚠️ CHALLENGE 2: Windows Filesystem Limitations  
Problem: Tokenization creates files with colons (timestamps), Windows forbids this
Solution: Switched to WSL Ubuntu for tokenization stage
Impact: Seamless Linux compatibility within Windows environment

⚠️ CHALLENGE 3: Missing Clinical Features (20+ errors)
Problem: Preprocessors expected labs, medications, demographic text - sample lacked these
Solution: Added defensive checks (if column exists → process, else skip) to 15+ functions
Impact: Pipeline handles limited datasets gracefully

⚠️ CHALLENGE 4: Vocabulary Gaps
Problem: Static patient codes (GENDER//M, GENDER//F) collected after vocabulary creation
Solution: Created fix_vocab.py to scan static data and expand vocabulary (69→72 tokens)
Impact: Training succeeded with complete token coverage

⚠️ CHALLENGE 5: Device Mismatch in Training
Problem: Model on GPU, validation data on CPU caused crashes
Solution: Modified metrics.py to explicitly move tensors to correct device
Impact: GPU training completed successfully

Key Takeaway: "Systematic debugging approach enabled pipeline completion despite 20+ technical hurdles"

SLIDE 8: POST-TOKENIZATION DATA PREPARATION
============================================
Title: "From Parquet to Safetensors: Creating Training-Ready Data"

Visual: Data transformation pipeline

Step 1: Load Tokenized Parquet Files
├─ train/0.parquet: 17,429 events, 356 patients
└─ test/0.parquet: 1,827 events, 39 patients

Step 2: Build Vocabulary (tensorize_data.py)
├─ Extract unique codes from train + test
├─ Add special tokens: <PAD>, <UNK>, <START>, <END>
└─ Assign integer IDs: 0-71 (72 total tokens)

Step 3: Fix Vocabulary Gaps (fix_vocab.py)
├─ Scan static_data.pickle for patient demographics
├─ Add missing codes: GENDER//M, GENDER//F, MEDS_BIRTH
└─ Final vocabulary: 72 tokens complete

Step 4: Merge Static Data (merge_static_data.py)
├─ Combine train and test patient demographics
└─ Output: 395 patients total

Step 5: Convert to Safetensors Format
├─ Patient timelines → Token ID sequences
├─ Example: [71, 69, 4, 23, 42, 11, 6, 28, 12, 8]
└─ Files: 0.safetensors (train), 1.safetensors (test)

Output Files:
✓ 0.safetensors - Train data (356 patients)
✓ 1.safetensors - Test data (39 patients)  
✓ vocab_t72.csv - Code-to-token mapping
✓ static_data.pickle - Patient demographics (395 total)
✓ interval_estimates.json - Temporal statistics

Why Safetensors? Fast loading, secure storage, efficient memory usage
